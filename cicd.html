GitHub CI/CD Setup Guide
Automate deployment with GitHub Actions and PM2
Overview
This guide will help you set up continuous integration and deployment (CI/CD) for your project using GitHub Actions. Follow these steps to automate deployment to your server whenever you push changes to the main branch.
Step 1: Initial Setup
First, deploy your project manually to the server and start it with PM2. Then go to your GitHub repository settings to add the required secrets.
1
SERVER_HOST - Your server IP address or hostname
2
SERVER_SSH_KEY - Your SSH private key for server access
Step 2: Create Workflow File
In your repository, create a file at .github/workflows/deploy.yml with the following content:
3
name: Deploy to server on: push: branches: [ main ] jobs: deploy: runs-on: ubuntu-latest steps: - name: Deploy to server uses: appleboy/ssh-action@v1 with: host: ${{ secrets.SERVER_HOST }} username: root key: ${{ secrets.SERVER_SSH_KEY }} port: 22 script: | export NVM_DIR="$HOME/.nvm" [ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh" cd /root/your-project git pull origin main npm install npx prisma generate npm run build pm2 restart your-project
Step 3: Test the Setup
Commit and push the workflow file to trigger the deployment process:
git add .
git commit -m "setup ci cd"
git push origin main
Important Note

For the first deployment, you need to manually pull the code on your VPS and start it with PM2. After that, the CI/CD pipeline will handle all future updates automatically.

Make sure to replace /root/your-project and your-project with your actual project path and PM2 process name.

Remember to keep your secrets secure and never expose them in your code.
